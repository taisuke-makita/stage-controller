# ステージコントローラ コマンド仕様

**対象バージョン:** stage_controller_firm_v0  
**更新日:** 2026-05-31

---

## 1. 通信仕様

| 項目 | 値 |
|------|-----|
| インタフェース | USB CDC (Virtual COM Port) |
| 終端文字（送信）| `\n` (LF, 0x0A) |
| 終端文字（受信）| `\n` (LF, 0x0A) |
| エンコーディング | ASCII |
| コマンドチェーン区切り | `;` |

コマンドは 1 行 1 コマンド（またはチェーン）を送信し、ファームウェアは 1 行の応答を返す。

---

## 2. 軸番号

| 番号 | 別名 | 説明 |
|------|------|------|
| `0` | `X` | 軸 0 (X) |
| `1` | `Y` | 軸 1 (Y) |
| `2` | `Z` | 軸 2 (Z) |

GUI の `axis_no()` 関数により `X`/`Y`/`Z` は `0`/`1`/`2` に変換される。ファームウェアは数字のみを受け付ける。

---

## 3. 応答フォーマット

| 種類 | 応答 | 説明 |
|------|------|------|
| 成功（値なし）| `OK\n` | 動作コマンド全般 |
| 成功（値あり）| `<値>\n` | クエリコマンド全般 |
| 複数行応答終端 | `END\n` | `RA` コマンドの最終行 |
| エラー | `NG\n` | パラメータ異常・実行不可 |

---

## 4. コマンド一覧

### 4.1 速度設定コマンド

速度コマンドは通常、移動コマンドとセミコロンでチェーンして使用する。

| コマンド | 方向 | 説明 | 応答例 |
|----------|------|------|--------|
| `MS<freq>` | → | パルス駆動周波数設定 [pps] | `OK` |
| `MSR` | ← | パルス駆動周波数読み出し | `1000` |
| `mS<speed>` | → | 物理量駆動速度設定 [mm/s 等] | `OK` |
| `mSR` | ← | 物理量駆動速度読み出し | `1.5` |

### 4.2 パルス量ベースの移動コマンド（`M` プレフィックス）

| コマンド | 説明 | 応答 | 備考 |
|----------|------|------|------|
| `MS<spd>;M<n>R<pls>` | 相対移動（即時応答）| `OK` | 大文字 R = 完了を待たず返答 |
| `MS<spd>;M<n>r<pls>` | 相対移動（完了待ち）| `OK` | 小文字 r = 完了後に返答 |
| `MS<spd>;M<n>A<pls>` | 絶対移動（即時応答）| `OK` | 大文字 A |
| `MS<spd>;M<n>a<pls>` | 絶対移動（完了待ち）| `OK` | 小文字 a |
| `M<n>P` | 現在位置読み出し [パルス] | `<pos>` | 例: `1234` |
| `M<n>I` | ホーミング（即時応答）| `OK` | |
| `M<n>i` | ホーミング（完了待ち）| `OK` | |
| `M<n>NP` | JOG 正方向（CW）開始 | `OK` | |
| `M<n>NM` | JOG 負方向（CCW）開始 | `OK` | |
| `M<n>F` | JOG 停止 | `OK` | |

- `<n>`: 軸番号 (0/1/2)
- `<pls>`: パルス数（正 = CW 方向、負 = CCW 方向）
- `<spd>`: 速度 [pps]

**送信例:**

```
MS1000;M0R500\n     → 軸0を+500パルス相対移動（即時）
MS500;M1r-200\n     → 軸1を-200パルス相対移動（完了待ち）
MS2000;M2A10000\n   → 軸2を絶対位置10000へ移動（即時）
M0P\n               → 軸0の現在位置を取得
M2NP\n              → 軸2のJOG正方向開始
M2F\n               → 軸2のJOG停止
M1i\n               → 軸1のホーミング（完了待ち）
```

### 4.3 物理量ベースの移動コマンド（`m` プレフィックス）

`M` を小文字 `m` にすると物理量（mm 等）での指定になる。内部でパルス数へ変換される（係数 = `P_motor_coeff[n]`）。

| コマンド | 説明 | 応答 |
|----------|------|------|
| `mS<spd>;m<n>R<dist>` | 相対移動（即時）[mm] | `OK` |
| `mS<spd>;m<n>r<dist>` | 相対移動（完了待ち）[mm] | `OK` |
| `mS<spd>;m<n>A<pos>` | 絶対移動（即時）[mm] | `OK` |
| `mS<spd>;m<n>a<pos>` | 絶対移動（完了待ち）[mm] | `OK` |
| `m<n>P` | 現在位置読み出し [mm] | `<pos>` |
| `m<n>I` / `m<n>i` | ホーミング（即時 / 完了待ち）| `OK` |
| `m<n>NP` / `m<n>NM` | JOG 正 / 負方向開始 | `OK` |
| `m<n>F` | JOG 停止 | `OK` |

**送信例:**

```
mS1.5;m0R2.0\n      → 軸0を+2.0mm相対移動（即時）
mS1.0;m1a-5.0\n     → 軸1を絶対位置-5.0mmへ移動（完了待ち）
m2P\n               → 軸2の現在位置を取得 [mm]
```

### 4.4 レジスタコマンド（`R` プレフィックス）

| コマンド | 説明 | 応答 |
|----------|------|------|
| `RS` | 全パラメータをフラッシュに保存 | `OK` |
| `RA` | 全パラメータ読み出し（複数行）| `<addr>:<name>:<unit>:<val>` × 256行 + `END` |
| `R<addr>S<val>` | 指定アドレスに値を書き込み（焼き込みなし）| `OK` |
| `R<addr>R` | 指定アドレスの値を読み出し | `<val>` |

- `<addr>`: レジスタアドレス (0〜255)
- `<val>`: 整数または浮動小数点数
- `<unit>`: 0=int, 1=float, 2=char

**`RA` 応答フォーマット（1 行あたり）:**

```
<addr>:<name>:<unit>:<val>\n
```

例:
```
0:firm_no   :0:1\n
1:box_no    :0:0\n
38:motor_freq:1:10.0\n
...
END\n
```

---

## 5. コマンドチェーン

セミコロン `;` で複数のサブコマンドを 1 回の送信にまとめられる。左から順番に実行され、最後に出力を持つサブコマンドの結果が応答として返る。

```
MS1000;M0R500\n
```
↑ `MS1000`（速度設定、応答なし）→ `M0R500`（移動、応答 `OK`）→ ファームが `OK\n` を返す

---

## 6. 動作の即時 / 完了待ち

移動コマンドは 2 種類の動作を持つ。

| 文字 | 動作 |
|------|------|
| 大文字 (`R`/`A`/`I`) | 即時応答：コマンドを受け付けてすぐ `OK` を返す。移動はバックグラウンドで実行。 |
| 小文字 (`r`/`a`/`i`) | 完了待ち：移動が完了してから `OK` を返す。GUI はこの間ブロックされる。 |

---

## 7. エラー応答

| 条件 | 応答 |
|------|------|
| 不明なコマンド | `NG\n` |
| 軸番号が範囲外 | `NG\n` |
| レジスタアドレスが範囲外 | `NG\n` |
| 物理量係数が未設定 (0) | `NG\n` |
| モーター動作中に移動コマンド | 無視（応答は `OK\n`）|
